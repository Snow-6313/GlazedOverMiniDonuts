How auth works

Username / password setup

  • The website should send the username and password to a secure backend login endpoint.
  • The backend verifies the password hash and returns a secure session or token.
  • The browser stores only the session/token, not the real password.
  • User creation, password resets, and roles should happen on the backend or auth provider.
Recommended later: Supabase Auth, Firebase Auth, or a custom backend using hashed passwords with bcrypt/Argon2.
